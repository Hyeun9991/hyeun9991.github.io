# 운영체제
## 운영체제란?
* **시스템의 자원과 동작을 관리하는 소프트웨어**
  * 시스템의 자원 : cpu, 메모리
  * 동작 : 프로그램 실행  
<br>

* **프로세스, 저장장치, 네트워킹, 사용자, 하드웨어를 관리**
  * 프로세스 : 실행하는 프로그램
  * 저장장치 : 메모리(RAM), HDD
  * 네트워킹
  * 사용자
  * 하드웨어

<br>

## 메모리 구조
#### 메모리 공간 종류
* `Code` : 소스 코드들이 들어가는 공간
* `Data` : 전역 변수, 정적 변수가 할당되는 공간
* `Heap` : 사용자가 직접 관리하는 영역으로 데이터가 동적으로 할당되는 공간
  * 런 타임에 크기가 결정됨 (사용중에 크기가 결정됨)
* `stack` : 함수의 호출정보, 지역변수, 매게변수들이 저장되는 공간
  * 컴파일 타임에 크기가 결정됨 (이미 메모리가 얼마나 차지할 지 결정되어 있음)

<div class="notice--info">
  <h4>프로그램을 실행할 때 메모리를 어떻게 사용할까?</h4>
  <br>
  사용자가 프로그램을 실행할 때 HDD는 프로그램의 정보를 읽어 메모리에 올리게되고<br>
  메모리에 올라온 명령들을 CPU가 실행됨
</div>

<br>

## 프로세스, 스레드
<div class="notice--info">
  유튜브 동영상으로 예시를 들자면<br>
  모바일이나 pc로 유튜브 동영상을 실행하는 것을 <b>프로세스</b>라고 하고<br>
  동영상 실행 프로그레스바와 좋아요 버튼을 클릭 시 올라가는 숫자들을 각각 <b>스레드</b>라고 함
  <br>  
  <b>즉, 운영체제에는 여러개의 프로세스가 존재하고, 프로세스에는 여러개의 스레드가 존재</b>
</div>

* 프로세스 : **실행중인 프로그램**
  * 각각의 **독립된** 메모리 영역을 할당받음 (code, data, stack, heap)

* 스레드 : **프로세스안에서 실행해주는 단위**
  * **프로세스 내에서 각각 stack만 따로 할당받고 code, data, heap 영역은 공유함**
  * 메서드의 역할이 각기 다르기 때문에 필요한 data만 공유하고 stack만 따로 할당받음

<br>

## CPU 스케줄러
* 준비큐에 있는 프로세스에 대해서 CPU를 할당하는 방법

> cpu는 하나의 프로그램만 실행할 수 있음
사용자가 유튜브를 보면서 카카오톡을 하고 있다고 가정했을 때<br>
cpu는 유튜브 프로그램을 잠깐 실행했다가 카카오톡을 실행함
그래서 동시에 실행되는 것처럼 보임

### 비선점 스케줄링
#### FCFS (First Come First Served)
* 먼저 cpu를 요청하는 프로세스를 먼저 처리하는 방식  
#### SJF (Shortest Job First)
* 전체 Burst Time이 **짧은** 프로세스를 먼저 처리하는 방식
* Burst Time : cpu에서 사용하는 시간

  |프로세스|Burst Time|
  |:------:|:--------:|
  |P1|6|
  |P2|3|
  |P3|8|
  |P4|7|
  
  버스트 시간이 짧은 순으로 실행이 됨
  P2 &rarr; P1 &rarr; P4 &rarr; P3

### 선점 스케줄링
#### SRT (Shortest Remaining Time)
* 진행 중인 프로스세가 있어도, **최단 잔여시간**인 프로세스를 먼저 처리하는 방식

  |프로세스|도착시간|Burst Time|
  |:------:|:----:|:--------:|
  |P1|0|6|
  |P2|1|3|
  |P3|2|8|
  |P4|3|7|

P1이 먼저 와서 실행되고 1초뒤에 P2가 왔을 때
P1의 버스트타임은 5초 P2의 버스트 타임은 3초가 되므로
P1이 아닌 P2를 실행
#### RR (Round Robin)
* 모든 프로세스가 같은 우선순위를 가지고, 같은 시간(타임 퀀텀)만큼 공평하게 실행되어 처리하는 방식
* 타임 퀀텀이 너무 크면 FCFS와 다를게 없고, 너무 작다면 불필요한 context switch가 많이 일어남 (적당하게 설정하는게 중요)
#### Priority Scheduling (우선 순위 스케줄링)
* 우선 순위가 높은 프로세스에 CPU를 우선 할당하는 방식의 스케줄링

<br>

## 가상 메모리
* 디스크를 효율적으로 사용하기 위해서 **사용하는 것만 메모리에 올리고** 나머지는 디스크에 보관
* cpu는 page table을 이용해서 가상 메모리에 접근할 수 있음

#### 정리
* 모든 프로세스에게 메모리를 할당하기에는 메모리의 크기가 한계가 있어서 사용하는 방법
* 프로세스에서 사용하는 부분만 메모리에 올리고 나머지는 디스크에 보관하는 기법을 가상 메모리라고 함

<br>

## 데드락 (Dead Rock)
* 프로세스가 자원을 얻지 못해 다음 작업을 못하는 상태

<div class="notice--info">
  P1,P2가 각각 자원 A와 B를 얻어야 하는데 P1은 A를, P2는 B를 가지고 있어서
  서로 무한정 기다리는 상태를 데드락이라고 함
</div>

### 데드락 발생 조건
아래 **네 가지 조건이 동시에** 성립할 때 발생함

* **상호 배제 (Mutual exclusion)**
  * 자원은 한 번에 한 프로세스만이 사용할 수 있어야 한다.
* **점유 대기 (Hold and wait)**
  * 하나의 자원을 점유하고 있으면서 다른 자원을 추가로 점유하기 위해 대기하고 있어야 한다.
* **비선점 (No preemption)**
  * 다른 프로세스에 할당된 자원은 사용이 끝날 때까지 강제로 빼앗아 올 수 없다.
* **순환 대기 (Circular wait)**
  * 자원을 점유하기 위해서 대기해야함을 순환해야한다.
