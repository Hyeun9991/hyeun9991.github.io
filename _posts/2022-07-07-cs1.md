## cpu : 계산하는 역할
* `ALU` : 수학적 계산, 산술 연산
* `레지스터`
* 메모리에 있는 명령들이 `ALU`와`레지스터`를 통해서 계산되어 실행됨
* cpu안에는 캐쉬 메모리가 있음

# 컴파일
개발자들은 소스나, 툴로 개발을 하면서 컴파일이라는 과정을 통해 실행 파일을 만들어냄
* `컴파일` : 마이너리 파일,실행할 수 있는 파일로 바뀌는 과정
  * `exe`
  * `dll` : 실행 파일

컴파일된 파일은 디스크에 저장되며 코드와 리소스로 구성되어 있음

# 프로그램(파일)의 실행과정

## 1. 디스크
컴파일된 파일의 일부분의 코드와 필요한 리소스가 메모리로 올라감

- - - 

## 2. 메모리
명령어들 중 일부, 처리해야 할 명령어가 cpu에 의해서 해당 명령어가 처리가 됨

### 프로세스
프로세스란, 파일로 저장되있던 프로그램이 메모리에 올라가는 순간을 프로세스라고 함

* `프로세서`와 `프로세스`
  * `프로세스` : 프로그램이 실행되고 있는 것
  * `프로세서` : cpu

- - - 

## 3. cpu
### 시분할
컴퓨터는 여러개의 프로세스를 실행 시키고 cpu는 여러개의 프로세스를 **시간**으로 나눠서 실행 시킴 이것을 `시분할` 이라고 함

### **스케줄링**
프로세스에게 얼마만큼 시간을 줄것이며, 어떤 프로세스를 먼저 실행할건지를 관리하는 것을 스케줄링이라고 함

#### Preemptive | 선점형 스케줄링
많은 os들이 살아남은 대표적인 스케줄링 방식 (윈도우,맥 등)

* `선점형` : 프로세스 중 **우선순위가 높은** 것을 먼저 cpu를 차지하는 방식

#### 그외의 스케줄링 방식
* `FCFS` : 먼저 들어온 요청부터 처리하는 방식
  * first come first served
* `FIFO`
  * first in first out
